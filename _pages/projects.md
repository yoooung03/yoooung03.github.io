## 🤖 FocusOn
**AI 행정 업무 자동화 서비스** · 코멘토 로컬AI메이커스랩 · 2025

![FocusOn](/assets/images/FocusOn.png)

코멘토 주관 기업 연계 일경험 프로그램에서 중소기업 대표의 행정 업무 비효율 문제를 정의하고, RAG 기반 LLM을 활용한 AI 행정 자동화 서비스를 기획·개발했습니다.

**문제 정의**
고객인 중소기업 대표님은 "핵심 업무에 집중할 시간이 부족하다"고 했습니다. 저는 그 말을 그대로 받아들이지 않고 업무 흐름을 직접 분석했습니다. 문서 확인과 승인이 6단계에 걸쳐 분산된 구조가 시간을 잡아먹고 있었고, 그 구조를 단순화하는 것이 진짜 해결책이었습니다.

---

**기획 과정**

- 사업화 멘토, 현업 대표 멘토와 함께 MVP 설계, 5Whys, 경쟁사 분석, 니즈 파악 진행
- 문제 파악에서 정의로 넘어가는 과정에서 "핵심 하나를 깊게 파야 한다"는 피드백 반복 수용
- 비전공자 기획자의 문제 정의를 기술적으로 검토하고 구체화하는 조력자 역할 수행
- 기획 완성 후 가설 검증을 위해 실제 고객 사용자 테스트 진행

---

**핵심 구현**

- RAG 기반 LLM을 활용한 대화형 문서 작성 가이드
- 자동 검수 및 요약 기능으로 문서 오류 사전 차단
- 원터치 승인/반려 기능으로 의사결정 흐름 단순화
- 승인 처리 대시보드 및 Todo 관리 시스템 구축
- Supabase DB와 Potens LLM API 연동으로 전체 서비스 구현

---

**나의 역할**

- 프론트엔드 개발 주담당 (Streamlit)
- LLM·DB 연동 기반 서비스 구현
- 비전공자 기획자의 문제 정의를 기술적으로 검토하고 구체화

---

**성과**

| 지표 | 결과 |
|------|------|
| 재작업 시간 | **80% 이상 절감** |
| 업무 착수 시간 | **20~30% 단축** |
| 핵심 업무 집중 비율 | **100% 향상** |
| 기존 업무 흐름 | **6단계 → 3단계** |

**기술 스택**
`RAG` `LLM` `Streamlit` `Supabase` `Potens API` `Python`

## 🩵 이음이
**AI 회상치료 서비스** · IBM AI 해커톤 → SideImpact → DataVenture · 2025

![이음이](/assets/images/eumm.png)

치매 노인을 위한 이미지 기반 AI 회상치료 서비스. 아이디어 제안부터 복지센터 현장 B2B 검증, 실제 배포 가능한 구조 완성까지 직접 수행했습니다.

**문제 정의**
기존 회상치료는 보호자나 복지사의 개입 없이 지속적인 사용이 어렵고, 대화 기반 정서치료가 전무한 상황이었습니다. 치매 환자가 사진을 보며 자연스럽게 기억을 떠올리고, AI와 대화하며 정서적 교감을 나눌 수 있는 서비스가 필요하다고 판단했습니다.

---

**1단계 · IBM AI 해커톤 (아이디어 → 프로토타입)**

- 서비스 아이디어를 직접 제안하고 디자인씽킹 기반 문제 정의 주도
- Streamlit 프론트엔드, MySQL DB 설계, LLM API 연동 전담
- 서비스 프로토타입 완성 → **대상 수상** 🏆

---

**2단계 · SideImpact (검증 → 개선 방향 도출)**

- 수상에 머물지 않고 실제 서비스 전환을 목표로 팀에 제안
- 복지센터를 직접 방문해 어르신 사용자 테스트 진행
- 원장님 B2B 인터뷰를 통해 치료 프로그램 구조, 비용 현실, 도입 의사결정 조건 확인
- 현장에서 발견한 두 가지 인사이트:
  - 이미지가 핵심인 서비스인데 어르신들이 눈이 침침해 이미지를 보지 못하심 → 음성 중심으로 재설계
  - AI 거부감을 예상했지만 오히려 빠르게 적응하고 긍정적 반응 → 진짜 장벽은 비용 구조
- **120개 팀 중 선정작** 공식 선정

---

**3단계 · DataVenture (재설계 → 운영 구조 완성)**

- 현장 피드백 기반으로 기존 Cloudflare 구조의 보안·확장성 한계 발견 → 전면 재설계 결정
- Spring Boot 백엔드 독자 설계, LLM 호출·대화 세션 관리 로직 구현
- Supabase DB 연동, AWS 배포, SSL 적용까지 전담
- 실제 배포 가능한 수준의 아키텍처 완성
- 결선에서 보안·시장성 피드백 직접 수용하며 기술 완성도와 사업성을 함께 고려하는 시각 확립

---

**성과**

| 단계 | 성과 |
|------|------|
| IBM AI 해커톤 | **대상 수상** 🏆 |
| SideImpact | **120개 팀 중 선정작** |
| DataVenture | 실제 배포 가능한 아키텍처 완성 |

**기술 스택**
`LLM` `Streamlit` `Spring Boot` `React` `AWS` `SSL` `Supabase` `MySQL` `Python`

## 📖 면접의 정석
**AI 면접 코칭 서비스** · 경상국립대학교 캡스톤 · 2025

![면접의 정석](/assets/images/myeonjub.png)

자기소개서 기반 AI 면접 질문 생성 및 비언어 피드백 제공 웹 서비스입니다. 취업 준비생이 실제 면접처럼 연습하고, AI 분석 결과를 PDF 리포트로 받아볼 수 있도록 설계했습니다.

**문제 정의**
면접 준비에서 가장 어려운 점은 자신의 자소서에서 어떤 질문이 나올지 예측하기 어렵고, 혼자 연습할 때 비언어적 표현을 객관적으로 파악하기 어렵다는 것입니다. AI로 이 두 가지를 동시에 해결하고자 했습니다.

---

**기획 과정**

- 서비스 전체 플로우 설계 주도 (자소서 입력 → 질문 생성 → 영상 녹화 → 분석 → 리포트)
- Figma로 전체 UI/UX 설계 후 React 기반으로 구현
- 팀 일정 지연 상황에서 전체 기능 흐름을 재정리하고 역할을 재분배해 기한 내 완성

---

**핵심 구현**

- 공개 자소서 30건 기반으로 생성형 AI를 활용해 약 10,000건의 질문 데이터셋 구축
- 출력 질문 수에 따른 가중치 설계를 적용해 KoBART 모델 파인튜닝
- 질문 간 TF-IDF 코사인 유사도 측정으로 모델 결과 검증
- React 기반으로 자소서 입력, 영상 녹화, 비언어 분석 결과 연동, PDF 리포트 생성까지 전체 사용자 플로우 구현
- Postman을 활용한 API 검증으로 백엔드와의 안정적인 연동 확보

---

**나의 역할**

- 프론트엔드 개발 주담당 (React)
- AI 모델 및 데이터셋 구축 (KoBART 파인튜닝)
- 전체 서비스 플로우 설계
- 팀 일정 지연 상황에서 기능 흐름 재정리 및 역할 재분배 주도

---

**성과**

| 지표 | 결과 |
|------|------|
| 데이터셋 | **10,000건 구축** |
| 학술 고도화 | **HCI Korea 2026 논문 발표** |
| 수상 | **경상국립대학교 소프트웨어 전시회 장려상** |

**기술 스택**
`KoBART` `React` `Python` `Figma` `Fine-tuning` `TF-IDF`

## 🐶 추견 60분
**AI 반려견 견종 분류 앱** · 2024

![추견 60분](/assets/images/dog.png)

실종 반려견 탐색을 돕는 견종 분류 모바일 앱입니다.

**문제 정의**
반려견 실종 시 견종을 빠르게 파악하기 어렵다는 문제에서 출발했습니다. 사진 한 장으로 견종을 분류하고 탐색을 돕는 서비스를 기획했습니다.

**핵심 구현**
- 웹 크롤링으로 견종별 1,500장 이상, 총 약 3만 장 데이터셋 직접 구축
- CNN, Swin Transformer, ResNet50, EfficientNet-B0, DenseNet201 비교 실험
- 데이터 기반으로 최적 모델 선정 후 React Native 기반 모바일 앱 구현

**성과**

| 지표 | 결과 |
|------|------|
| 분류 정확도 | **93.6%** |
| 비교 실험 모델 수 | **5개** |
| 구축 데이터셋 | **약 3만 장** |

**기술 스택**
`PyTorch` `DenseNet201` `React Native` `Spring Boot` `Python`

## 🚗 중고차 가격 예측
**ML 기반 가격 예측 서비스** · USG 공유대학 · 2024

![중고차 가격 예측](/assets/images/car.png)

Kaggle 중고차 데이터 기반 머신러닝 가격 예측 서비스입니다.

**문제 정의**
중고차 가격은 다양한 변수에 의해 결정되지만 기준이 불명확해 소비자가 적정 가격을 판단하기 어렵습니다. 데이터 기반으로 가격을 예측하는 서비스를 개발했습니다.

**핵심 구현**
- IQR 기반 이상치 제거, 로그 변환 등 체계적인 데이터 전처리 파이프라인 구축
- CatBoost, LightGBM, XGBoost, Random Forest 비교 분석 후 최적 모델 선정
- Streamlit 기반 예측 UI 개발로 결과 시각화

**성과**

| 지표 | 결과 |
|------|------|
| 최종 모델 | **CatBoost** |
| R² 정확도 | **93.5%** |
| 비교 실험 모델 수 | **5개** |

**기술 스택**
`Python` `CatBoost` `Streamlit` `Scikit-learn` `Pandas`
